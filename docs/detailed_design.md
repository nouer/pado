## 3. バリデーション仕様

### 3.1 取引先バリデーション (`validatePartner`)

| フィールド | ルール | エラーメッセージ |
|-----------|--------|----------------|
| name | 必須、1〜100文字 | 「取引先名を入力してください」/「取引先名は100文字以内で入力してください」 |
| nameKana | 任意、ひらがな+長音記号+スペース、100文字以内 | 「ふりがなはひらがなで入力してください」/「ふりがなは100文字以内で入力してください」 |
| partnerType | 必須、customer/supplier/both | 「取引先区分を選択してください」 |
| phone | 任意、半角数字+ハイフン、7〜15文字 | 「電話番号は半角数字とハイフンで入力してください」/「電話番号は7〜15文字で入力してください」 |
| fax | 任意、半角数字+ハイフン、7〜15文字 | 「FAX番号は半角数字とハイフンで入力してください」 |
| email | 任意、email形式 | 「メールアドレスの形式が不正です」 |
| postalCode | 任意、半角数字+ハイフン、7〜8文字 | 「郵便番号の形式が不正です」 |
| invoiceRegistrationNumber | 任意、T+13桁数字 | 「登録番号はT+13桁の数字で入力してください」 |

### 3.3 帳票バリデーション (`validateDocument`)

| フィールド | ルール | エラーメッセージ |

| フィールド | ルール | エラーメッセージ |
|-----------|--------|----------------|
| businessName | 必須、1〜100文字 | 「事業者名を入力してください」 |

### 3.6 インボイス登録番号バリデーション (`validateInvoiceNumber`)

```
パターン: /^T\d{13}$/
- 先頭1文字が "T"（大文字固定）
- 続く13文字が半角数字
- 合計14文字
```

## 4. 帳票編集オーバーレイ詳細

### 4.1 セクション1: 基本情報カード

| 要素 | 型 | 挙動 |
|------|-----|------|
| 帳票番号 | readonly text | 新規作成時は自動採番、編集時は変更不可 |
| 発行日 | date input | デフォルト: 当日。変更可能 |
| ステータス | select | draft/issued/sent/paid/cancelled |
| 取引先 | select + 検索 | 取引先マスタから選択。選択時に名前を表示 |

### 4.2 セクション2: 種別固有フィールド

帳票種別に応じて表示するフィールドを切り替える。全ての種別固有フィールドは任意（空でも保存可能）。

| 帳票種別 | 表示フィールド |
|---------|--------------|
| 見積書 (estimate) | 有効期限（date、デフォルト: 発行日+30日） |
| 発注書 (purchase_order) | 納品希望日（date） |
| 請求書 (invoice) | 支払期限（date、デフォルト: 発行月末日） |
| 納品書 (delivery_note) | 納品日（date、デフォルト: 当日） |
| 売上伝票 (sales_slip) | 摘要（text） |
| 仕入伝票 (purchase_slip) | 摘要（text） |
| 領収書 (receipt) | 但し書き（text、デフォルト:「お品代として」） |

### 4.3 セクション3: 明細行エリア

#### 明細行テーブル

| 列 | 幅目安 | 入力型 | 挙動 |
|----|-------|--------|------|
| No | 40px | readonly | 自動連番（1, 2, 3...） |
| 品目名 | flex | select + text | 品目マスタからドロップダウン選択。選択時にデフォルト単価・単位・税区分を自動入力。自由入力も可能 |
| 数量 | 80px | number | step="1"、min="0" |
| 単位 | 60px | text | 品目選択時に自動入力、変更可能 |
| 単価 | 100px | number | 品目選択時に自動入力、変更可能 |
| 税区分 | 80px | select | 10% / 8% / 対象外 |
| 金額 | 100px | readonly | 数量 x 単価（自動計算） |
| 削除 | 40px | button | 行削除ボタン（明細行が1行のみの場合は無効） |

#### 動的行操作

- 「＋行追加」ボタンで末尾に空行を追加
- 各行の削除ボタンで行を削除（最低1行は残す）
- 品目選択時のオートフィル: 単価・単位・税区分を品目マスタから自動入力

### 4.4 セクション4: 金額集計エリア

| 表示項目 | 計算方法 |
|---------|---------|
| 小計 | 全明細行の金額合計（税抜） |
| 10%対象額 | taxRateType="rate10" の明細行金額合計 |
| 消費税（10%） | 計算設定に基づく（明細行ごと or 合計に対して、端数処理） |
| 8%対象額 | taxRateType="rate8" の明細行金額合計 |
| 消費税（8%） | 計算設定に基づく |
| 対象外額 | taxRateType="exempt" の明細行金額合計 |
| 合計金額 | 小計 + 消費税（10%）+ 消費税（8%） |
| 収入印紙注記 | 領収書のみ。税抜金額に基づく印紙税額を表示 |

リアルタイム計算: 数量・単価・税区分の変更時に即座に再計算して表示する。

## 7. 設定画面詳細

### 7.1 発行者情報セクション

フォームフィールド: 事業者名（屋号）、代表者名、郵便番号、住所、電話番号、FAX番号、メールアドレス、登録番号（T+13桁）、振込先情報（複数行テキスト）

### 7.2 計算設定セクション

| 設定項目 | UI | 選択肢 |
|---------|-----|--------|
| 消費税計算方式 | ラジオボタン | 明細行ごとに計算 / 合計に対して計算 |
| 端数処理 | ラジオボタン | 切り捨て / 四捨五入 / 切り上げ |

### 7.3 データ管理セクション

| 機能 | ボタンラベル | 挙動 |
|------|------------|------|
| エクスポート | データをエクスポート | 全ストアデータをJSONでダウンロード |
| インポート | データをインポート | JSONファイルをアップロードして復元 |
| 全データ削除 | 全データを削除 | 確認ダイアログ(2段階)後に全ストアクリア |
| サンプルデータ | サンプルデータを読み込む | sample_data.json をインポート |
| 自動更新確認 | — | 30秒間隔の自動チェック + ページ可視性変更時チェック |

#### 初回アクセス時のサンプルデータ自動ロード

- **判定条件**: `documents` ストアと `partners` ストアが両方とも空の場合に初回アクセスと判定
- **タイミング**: `DOMContentLoaded` 内の `loadDocList()` 実行後に判定
- **動作**: `sample_data.json` を自動取得し、確認ダイアログなしでサイレントにインポート
- **重複チェック**: 手動インポートと同様、同一IDのデータはスキップ
- **案内表示**: インポート完了後、消えない `info` タイプのトーストで以下のメッセージを表示:
  - 「サンプルデータを読み込みました。ご自身のデータで始める場合は、設定タブの「全データ削除」からリセットできます。」
- **再表示なし**: データが存在する状態でのリロードではトーストは表示されない

### 7.4 表示設定セクション

設定画面に「表示設定」セクションを設け、各帳票種別の表示/非表示をトグルで設定する。

| 項目 | 仕様 |
|------|------|
| 対象 | 領収書を除く6種の帳票種別 |
| UI | トグルスイッチ |
| 保存先 | `hiddenDocTypes` として IndexedDB に保存 |
| 効果 | 非表示に設定した帳票種別のタブが帳票一覧に表示されなくなる |

## 8. 帳票変換ダイアログ詳細

### 8.1 変換先選択UI

変換元の帳票種別に応じて、変換可能な帳票種別をボタンで表示する。

| 変換元 | 表示される変換先ボタン |
|-------|---------------------|
| 見積書 | 請求書 / 発注書 / 納品書 |
| 発注書 | 仕入伝票 |
| 納品書 | 売上伝票 |
| 請求書 | 領収書 |
| 売上伝票 | 請求書 |
| 仕入伝票 | (変換不可) |
| 領収書 | (変換不可) |

### 8.2 変換処理フロー

```
1. ユーザーが変換先帳票種別を選択
2. 変換先の帳票番号を自動採番
3. 元帳票のデータをコピー:
   - 取引先情報
   - 明細行（全行）
   - 備考
4. 変換先固有フィールドにデフォルト値を設定:
   - 発行日 = 当日
   - 種別固有フィールド = デフォルト値
   - ステータス = draft
5. sourceDocId に元帳票IDを設定
6. 新規帳票として保存
7. 変換先帳票の編集オーバーレイを表示
```

## 10. ステータス遷移

### 10.1 ステータス定義

| ステータス | ラベル | 説明 |
|-----------|--------|------|
| `draft` | 下書き | 作成中・未発行 |
| `issued` | 発行済み | 発行完了 |
| `sent` | 送付済み | 取引先に送付済み |
| `paid` | 入金済み | 代金回収完了（請求書・領収書） |
| `cancelled` | 取消 | 取消済み |

### 10.2 遷移ルール

ステータスは select による手動変更とする。自動遷移は行わない。全てのステータス間で自由に遷移可能（業務上の柔軟性を優先）。

## 11. 検索仕様

### 11.1 帳票検索

- 検索対象: 帳票番号（docNumber）、取引先名（partnerName）
- 検索方式: 部分一致（大文字小文字区別なし）
- 検索タイミング: 入力のたびにリアルタイム検索（debounce 300ms）

### 11.2 取引先検索

- 検索対象: 取引先名（name）、ふりがな（nameKana）、取引先コード（partnerCode）
- 検索方式: 部分一致（大文字小文字区別なし）

### 11.3 品目検索

- 検索対象: 品目名（name）、品目コード（itemCode）
- 検索方式: 部分一致（大文字小文字区別なし）
