# 要件定義書 — パド帳票管理 (pado)

## 1. はじめに

### 1.1 目的

本書は、小規模事業者向けブラウザ完結型帳票管理アプリケーション「パド帳票管理（pado）」の要件を定義する。

### 1.2 背景

小規模事業者・個人事業主においては、以下の課題がある。

- 見積書・請求書等の作成に表計算ソフトや手書きを使用しており、管理が煩雑
- インボイス制度（適格請求書等保存方式）への対応が負担となっている
- 帳票間の変換（見積書から請求書へ等）が手作業で非効率
- 収入印紙の要否判断に不安がある
- 既存のクラウド帳票サービスは月額費用が障壁となる

### 1.3 対象ユーザー

- 個人事業主・フリーランス
- スタッフ1〜5名程度の小規模事業者
- 適格請求書発行事業者
- ITリテラシーが標準的な事業者

---

## 2. システム概要

ブラウザ上で動作する SPA（Single Page Application）として実装し、すべてのデータをブラウザ内の IndexedDB に保存する。サーバーサイドの処理は不要であり、静的ファイルの配信のみで運用可能とする。

- **動作環境**: モダンブラウザ（Chrome, Firefox, Safari, Edge）
- **データ保存先**: ブラウザ内 IndexedDB
- **ネットワーク**: オフラインでも完全動作（PWA対応）
- **デプロイ**: Docker + nginx による静的ファイル配信
- **印刷**: ブラウザの印刷機能のみ使用（CSS print stylesheet）

## 3. 機能要件

### 3.1 帳票管理機能

#### 対応帳票（7種類）

| # | 帳票種別 | 帳票コード | 番号プレフィックス | 概要 |
|---|---------|-----------|-------------------|------|
| 1 | 見積書 | estimate | EST | 取引先への見積金額を提示 |
| 2 | 発注書 | purchase_order | PO | 仕入先への発注内容を通知 |
| 3 | 請求書 | invoice | INV | 取引先への代金請求（インボイス対応必須） |
| 4 | 納品書 | delivery_note | DN | 商品・サービスの納品を通知 |
| 5 | 売上伝票 | sales_slip | SS | 売上の社内記録 |
| 6 | 仕入伝票 | purchase_slip | PS | 仕入の社内記録 |
| 7 | 領収書 | receipt | RCP | 代金受領の証明（収入印紙対応必須） |

#### 帳票共通項目

| フィールド | 型 | 必須 | バリデーション |
|-----------|-----|------|-------------|
| 帳票番号 | string | ○ | 自動採番（プレフィックス-年-連番4桁） |
| 発行日 | date | | デフォルト: 当日 |
| ステータス | enum | ○ | draft / issued / sent / paid / cancelled |
| 取引先 | ref | | 取引先マスタから選択 |
| 明細行 | array | ○ | 1行以上必須 |
| 備考 | string | | 2000文字以内 |
| 発行者情報 | object | ○ | 設定タブから読み取り（自動反映） |

#### 明細行項目

| フィールド | 型 | 必須 | バリデーション |
|-----------|-----|------|-------------|
| 品目 | ref/string | ○ | 品目マスタから選択 or 自由入力 |
| 数量 | number | ○ | 0以上、小数第2位まで |
| 単位 | string | | 例: 個、式、時間、kg |
| 単価 | number | ○ | 0以上、整数 |
| 税区分 | enum | ○ | 10% / 8%（軽減税率） / 対象外 |
| 金額 | number | - | 自動計算（数量 x 単価） |

#### 帳票種別固有項目

| 帳票種別 | 固有フィールド | 型 | 備考 |
|---------|--------------|-----|------|
| 見積書 | 有効期限 | date | 発行日から30日がデフォルト |
| 発注書 | 納品希望日 | date | |
| 請求書 | 支払期限 | date | 月末がデフォルト |
| 納品書 | 納品日 | date | デフォルト: 当日 |
| 売上伝票 | 摘要 | string | |
| 仕入伝票 | 摘要 | string | |
| 領収書 | 但し書き | string | デフォルト: 「お品代として」 |

#### 帳票機能

- 帳票の新規作成・編集・削除
- 帳票番号の自動採番（例: INV-2026-0001）
- 帳票一覧表示（種別タブで切り替え）
- 帳票検索（帳票番号・取引先名で部分一致）
- 帳票の複製
- 帳票間の変換（見積書→請求書、見積書→発注書 等）
- ブラウザ印刷（CSS print stylesheet によるレイアウト）

### 3.2 インボイス制度（適格請求書等保存方式）対応

#### 必須記載事項

| # | 記載事項 | 実装方法 |
|---|---------|---------|
| 1 | 適格請求書発行事業者の氏名又は名称 | 設定タブの発行者情報から自動反映 |
| 2 | 登録番号（T+13桁） | 設定タブで入力、帳票に自動印字 |
| 3 | 取引年月日 | 発行日フィールド |
| 4 | 取引内容 | 明細行の品目名 |
| 5 | 税率ごとに区分した対価の額（税抜又は税込） | 金額集計エリアで税率別に表示 |
| 6 | 税率ごとに区分した消費税額 | 金額集計エリアで税率別に表示 |
| 7 | 書類の交付を受ける事業者の氏名又は名称 | 取引先名フィールド |

#### 登録番号バリデーション

- 形式: `T` + 数字13桁（例: T1234567890123）
- 入力時にフォーマットチェック
- 未入力の場合は帳票に「登録番号未設定」と警告表示

### 3.3 収入印紙対応

#### 対象帳票

- 領収書（金銭又は有価証券の受取書）

#### 収入印紙判定基準

- **判定基準額**: 税抜金額（消費税額が明記されている場合）
- **非課税境界**: 税抜50,000円未満は非課税（収入印紙不要）
- **領収書への表示**: 収入印紙の要否・金額を帳票上に注記

#### 収入印紙税額表（税抜金額基準）

| 税抜金額 | 印紙税額 |
|---------|---------|
| 5万円未満 | 非課税（不要） |
| 5万円以上〜100万円以下 | 200円 |
| 100万円超〜200万円以下 | 400円 |
| 200万円超〜300万円以下 | 600円 |
| 300万円超〜500万円以下 | 1,000円 |
| 500万円超〜1,000万円以下 | 2,000円 |
| 1,000万円超〜3,000万円以下 | 6,000円 |
| 3,000万円超〜5,000万円以下 | 10,000円 |
| 5,000万円超〜1億円以下 | 20,000円 |
| 1億円超〜2億円以下 | 40,000円 |
| 2億円超〜3億円以下 | 60,000円 |
| 3億円超〜5億円以下 | 100,000円 |
| 5億円超 | 200,000円 |

### 3.6 帳票変換機能

#### 変換パターン

| 変換元 | 変換先 | 用途 |
|-------|-------|------|
| 見積書 | 請求書 | 見積もりが承認された後の請求 |
| 見積書 | 発注書 | 見積もり内容での発注 |
| 見積書 | 納品書 | 見積もり内容での納品 |
| 発注書 | 仕入伝票 | 発注に対する仕入計上 |
| 納品書 | 売上伝票 | 納品に対する売上計上 |
| 請求書 | 領収書 | 入金確認後の領収書発行 |
| 売上伝票 | 請求書 | 売上に対する請求 |

#### 変換仕様

- 変換先は新規帳票として作成（元の帳票は変更しない）
- 明細行・取引先・備考はコピーされる
- 帳票番号は変換先の種別で新規採番
- 変換元帳票IDを `sourceDocId` として保持（トレーサビリティ）
- 発行日は変換実行日がデフォルト

### 3.7 印刷機能

#### 印刷方式

- ブラウザ標準の印刷機能（`window.print()`）を使用
- CSS print stylesheet でレイアウトを制御
- 外部ライブラリ不使用

#### 印刷レイアウト（共通）

- 用紙サイズ: A4縦
- 帳票種別名（タイトル）
- 発行者情報（右上）
- 取引先情報（左上）
- 帳票番号・発行日
- 明細テーブル（品目・数量・単位・単価・税区分・金額）
- 税率別内訳（10%対象額・8%対象額・消費税額）
- 合計金額
- 備考
- 登録番号（インボイス対応帳票）

#### 領収書レイアウト

- コンパクトレイアウト（手書き風スタイル）
- 収入印紙貼付エリア（印紙が必要な場合に枠線表示）
- 印紙税額の注記

---

### 3.8 データエクスポート/インポート機能

#### エクスポート

- 全データ（取引先・品目・帳票・連番・設定）をJSON形式でダウンロード
- ファイル名: `pado_backup_YYYYMMDD_HHMMSS.json`

#### インポート

- JSONファイルをアップロードしてデータを復元
- `appName: 'pado'` によるフォーマット検証
- 既存データとのマージ（同一IDはスキップ）

#### 全データ削除

- 確認ダイアログ表示後、全ストアをクリア

#### 初回アクセス時のサンプルデータ自動読み込み

- 初回アクセス時（documents・partnersストアが共に空の場合）にサンプルデータを自動ロード
- 確認ダイアログなしでサイレントにインポートし、完了後に消えない案内トーストを表示
- ユーザーはトーストの案内に従い、設定タブの「全データ削除」でリセット可能

## 4. 非機能要件

### 4.1 パフォーマンス

- 取引先100件、品目500件、帳票10,000件でもスムーズに動作すること
- 帳票一覧の表示・検索が1秒以内に完了すること

### 4.2 セキュリティ

- HTML特殊文字は必ず `escapeHtml()` でエスケープし、XSSを防止すること
- データはブラウザ内に保存し、外部サーバーへの送信は一切行わないこと

### 4.3 ユーザビリティ

- レスポンシブデザイン（モバイル375px〜デスクトップ1280px+）
- 帳票種別の切り替えがタブ操作で直感的に行えること
- 金額の自動計算・税率別集計がリアルタイムで表示されること

### 4.4 可用性

- オフライン環境でも全機能が利用可能であること
- ブラウザのみで動作し、プラグインやネイティブアプリが不要であること

### 4.5 データ保全

- JSON形式によるエクスポート/インポートでデータバックアップが可能であること
- インポート時にデータ形式を検証し、不正データを拒否すること

### 4.6 互換性

- Chrome, Firefox, Safari, Edge の最新版で動作すること
- iOS Safari での PWA 動作に対応すること

### 4.7 法令準拠

- インボイス制度（適格請求書等保存方式）の記載要件を満たすこと
- 収入印紙税額表（印紙税法別表第一の第17号文書）に準拠すること
